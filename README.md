# Desafio: Programação Orientada a Objetos
### Projeto prático desenvolvido durante o Bootcamp Java da [Digital Innovation One](https://web.digitalinnovation.one/) 🚀

Neste projeto, tive a oportunidade de aplicar os pilares fundamentais da Programação Orientada a Objetos (POO) em um cenário real: a modelagem de um ecossistema de Bootcamp. O objetivo foi transformar conceitos teóricos em um sistema funcional e bem estruturado.

---

## 🎯 Objetivo do Projeto
Colocar em prática a lógica de negócio de uma plataforma de cursos, utilizando Java para exercitar:
* **Abstração:** Focar nos atributos essenciais (Cursos, Mentorias e Devs).
* **Encapsulamento:** Proteger os dados e a lógica interna das classes.
* **Herança:** Criar uma estrutura eficiente onde Curso e Mentoria herdam de uma classe base (Conteúdo).
* **Polimorfismo:** Permitir comportamentos diferentes para o cálculo de XP, dependendo do tipo de conteúdo.

---

## 💻 Tecnologias e Ferramentas Utilizadas
* **Linguagem:** Java 17 (ou superior)
* **IDE:** VS Code
* **Versionamento:** Git & GitHub
* **Paradigma:** Programação Orientada a Objetos (POO)

---

## 👣 Evolução do Desenvolvimento
1.  **Modelagem do Domínio:** Análise do problema e definição das entidades principais.
2.  **Criação das Classes:** Implementação das classes `Bootcamp`, `Curso`, `Mentoria` e `Dev`.
3.  **Aplicação de POO:**
    * Uso de **Herança** para evitar repetição de código entre Cursos e Mentorias.
    * Uso de **Collections** (como `Set` e `LinkedHashSet`) para gerenciar inscritos e conteúdos.
4.  **Instanciação:** Criação de objetos reais para simular a interação de desenvolvedores com o Bootcamp.

---

## 📚 Conceitos que Consolidei
Durante a execução, reforcei minha base teórica sobre:
* **Domínio:** A camada de negócio onde as regras acontecem.
* **Classes e Objetos:** A diferença entre o "molde" e o "produto final".
* **Métodos e Atributos:** Como definir comportamentos e estados.
* **Instanciação:** O momento em que o código ganha vida na memória usando o operador `new`.
